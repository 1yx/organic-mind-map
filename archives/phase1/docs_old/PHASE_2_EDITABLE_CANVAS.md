# Phase 2: 可编辑画布 — 从生成图中提取素材

## 1. 背景

Phase 1 已完成从 YAML 结构到只读 SVG 渲染的完整管线。Phase 2 的目标是让用户能**编辑**由 GPT Image 2 生成的手绘思维导图。

核心问题：GPT Image 2 生成的是一张完整位图，用户无法修改其中的文字、移动分支或替换插画。要让生成图变得可编辑，需要将位图分解为可操作的独立素材。

## 2. 素材提取管线

### 2.1 管线总览

```
YAML 结构 ──▶ GPT Image 2 生成 ──▶ 合成图 PNG
                                     │
                 ┌───────────────────┤
                 ▼                   ▼
          视觉模型检测              Canvas 像素处理
          返回 bounding box         去除标记色像素
                 │                   │
                 └───────┬───────────┘
                         ▼
                  透明背景插画素材
                         │
                         ▼
                  贴到可编辑 SVG 画布
```

### 2.2 生成阶段：色度键（Chroma Key）标记

**关键决策**：在 prompt 中要求 GPT Image 2 使用特定颜色绘制分支线条和边框，使这些元素可以被程序精确识别和移除。

**Prompt 约束**：

- 透明背景（无纸质纹理）
- 分支线条使用特定的标记色（如纯品红 `#FF00FF`）
- 分支边框使用特定的标记色（如纯青色 `#00FFFF`）
- 插画保持自然色彩

**为什么标记色比"去纸张底色"靠谱**：标记色是纯色、高饱和度，RGB 阈值判断几乎零误判。纸张底色有纹理、不均匀，阈值很难调。透明背景进一步消除了底色问题。

### 2.3 检测阶段：视觉模型定位

使用视觉模型（GPT-4o / Claude Vision）检测合成图中每个插画的 bounding box。

**输入**：合成图 PNG + YAML 结构（用于匹配插画到对应的分支/子节点）

**输出**：像素坐标格式的 bounding box 列表

```json
{
  "illustrations": [
    {
      "branch": "极速交付",
      "child": "研究预览",
      "bbox": { "x": 340, "y": 220, "width": 120, "height": 110 }
    }
  ]
}
```

**为什么选择视觉模型而非传统 CV**：

- 生成图的布局由图像模型决定，每次不同，无法用固定模板定位
- 传统 CV（连通域分析）对不同风格脆弱
- 视觉模型理解语义，能区分"插画"和"装饰性曲线"

### 2.4 裁切与清理阶段：Canvas 像素处理

在浏览器中使用 Canvas API 处理裁切区域：

```js
// 伪代码
for (each pixel in cropped region) {
  if (matchesChromaKey(pixel, chromaColors)) {
    pixel.alpha = 0;  // 标记色 → 透明
  }
  // 其他像素保留（就是插画内容，背景已经是透明的）
}
```

**标记色匹配**：精确 RGB 匹配 + 小容差（处理抗锯齿边缘）

**结果**：透明背景的插画素材，只包含插画本身的像素

### 2.5 放置阶段：SVG 画布

将处理后的透明素材作为 SVG `<image>` 元素放置到可编辑画布上：

```svg
<image href="data:image/png;base64,..." x="340" y="220" width="120" height="110" />
```

素材继承可编辑画布的交互能力：选择、拖拽、缩放。

## 3. 关键设计决策

### 3.1 为什么不单独生成每张插画

生成式 AI（GPT Image 2）在不同调用间无法保证风格一致性。分开生成 12 张插画会产生 12 种不同的画风，视觉上无法统一。

**结论**：必须在一次生成中产出所有插画，然后从合成图中裁切提取。

### 3.2 为什么不用传统图像分割

- 纸张纹理不均匀，阈值法不可靠
- 分支线条穿过插画区域，连通域分析会误切
- 不同生成风格需要重新调参

**结论**：用标记色 + 视觉模型定位的组合方案，兼顾精确性和通用性。

### 3.3 透明背景 vs 纸质纹理

选择透明背景。虽然纸质纹理在视觉上有手绘感，但：

- 透明背景使像素处理逻辑极简
- 纸质纹理可以在画布层面统一添加（作为 SVG 底层 pattern）
- 避免了最难的"去不均匀底色"问题

## 4. 待验证问题

- **标记色会不会影响生成质量**：GPT Image 2 在约束分支颜色后，插画质量是否会下降？需要 A/B 测试。
- **视觉模型定位精度**：bounding box 的偏差有多大？是否需要二次校准？
- **抗锯齿边缘处理**：标记色和插画色交界处的混合像素怎么处理？可能需要 alpha 渐变而非二值判断。
- **标记色与插画色的冲突**：如果插画本身包含接近标记色的颜色（如品红色花朵），如何避免误切？

## 5. 与 Phase 1 的关系

Phase 1 的 `editable-handdraw-canvas-demo` 已实现：
- 手动映射的 SVG 画布
- 选择、拖拽、文字编辑
- JSON 状态导入/导出
- 参考图叠加

Phase 2 将在此基础上：
- 用提取的真实插画替换 SVG 占位符 doodle
- 加入自动化的素材提取管线
- 支持任意生成图（不限于 mindmap-6 fixture）
