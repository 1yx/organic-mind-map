# GPT-Image-2 HEX 颜色码遵守稳定性调研

> 调研日期：2026-05-04

## 结论

**GPT-Image-2 无法 100% 精确匹配纯 HEX 颜色码。** 模型生成的是"视觉上接近"的颜色，像素级精确匹配不可实现。

## 证据来源

### 1. 学术论文 "Close, Not Exact: Brand-Colour Fidelity in GPT-4o Image Generation"

- 研究了 GPT 图像生成模型对品牌色的精确还原能力
- 核心发现：模型生成的是"视觉上接近"的颜色，但像素级精确匹配不可实现
- HEX 值会有偏移，偏差范围通常在肉眼可接受的 ΔE 范围内

### 2. GenColorBench 基准测试 (arxiv 2510.20586)

- 44K 条颜色提示词的大规模评测
- 关键数据：**数值型颜色（HEX/RGB）的准确率显著低于语义颜色名称**
  - 语义名称如 "red", "sky blue" → 准确率较高
  - HEX 码如 `#FF5733` → 准确率明显下降
- 所有 T2I 模型（包括 GPT-Image 系列）都存在这个问题

### 3. 社区实测（@ZHO_ZHO_ZHO 的 X/Twitter 对比测试）

- GPT-Image-2 的颜色还原能力 ≈ GPT-Image-Pro 水平
- 单色场景表现尚可，多色场景偏差增大
- 仍然不是 100% 精确

### 4. OpenAI 官方提示指南

- 官方文档中**没有专门针对 HEX 颜色控制的指导**
- 精确颜色控制并非该模型的设计目标

## 影响因素

| 因素 | 影响 |
|------|------|
| 颜色数量 | 1-2 色偏差小，3-4 色以上偏差明显增大 |
| 色差距离 | 相近色（如 `#FF0000` vs `#FF3300`）更容易混淆 |
| 复杂场景 | 背景色比前景主体色更难精确控制 |
| 颜色格式 | 语义名 > HEX 码（与直觉相反） |

## 最佳实践（提高颜色稳定性）

1. **HEX + 语义名称双写**：`"background color #FF5733 (vivid orange-red)"`
2. **限制颜色数量**：单张图控制在 3-4 色以内
3. **使用强调词**：`"exactly"`, `"precisely"`, `"must be"` 等关键词可略微提升遵守度
4. **避免相近色**：色差太小的颜色组合更容易被混淆
5. **后处理兜底**：如果需要像素级精确，考虑用程序化后处理替换颜色区域

## 实际建议

如果需要精确的颜色输出（如品牌配色、UI 设计稿），**不要依赖 gpt-image-2 做 100% HEX 精确匹配**。可行策略：

- 用 GPT-Image-2 生成结构和内容，颜色作为"近似参考"
- 关键颜色区域通过后处理程序替换为精确 HEX
- 或者用语义化描述代替纯 HEX，反而可能更稳定
