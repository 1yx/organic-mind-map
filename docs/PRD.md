# PRD: Organic Mind Map (有机思维导图)

## 1. 产品背景
在思维导图领域，Tony Buzan 的“有机思维导图”以其发散性、色彩丰富和曲线美学著称。目前的市场标杆是 Ayoa (Organic Mode)，但其闭源且订阅费用昂贵。
本项目旨在创建一个**开源、高性能、高可定制化**的有机思维导图工具，不仅提供极致的视觉还原，更通过“规则约束优先 + 意图式编辑”的方式确保产出严格符合博赞有机思维导图法则，而不是退化为普通树状脑图、文字提纲或无限画布白板。

## 1.1 MVP 使用场景

MVP 不以可视化编辑器为中心，而是先验证“长文本到合格有机导图渲染”的最短链路：

1. 用户在 Gemini CLI / Codex CLI / Claude Code 等 Agent CLI 中调用本项目的 agent skill。
2. Agent skill 调用大模型，将长文本按内容切成符合博赞要求的简短层级列表。
3. Agent CLI 将列表传给本项目 CLI。
4. 本项目 CLI 做契约校验和容量阈值检查；若内容过多，返回可被 Agent 捕获的错误，要求 Agent 重新生成更精简的列表。
5. 校验通过后，CLI 启动本地 Web 服务。
6. 浏览器端基于真实 DOM/SVG 文本测量完成布局和预览。
7. 用户在本地 Web 页面查看有机导图样式，并导出 `.omm` 与 PNG 图片。

MVP 暂不实现 `.omm` 的可视化编辑功能。节点编辑、拖拽重组、撤销重做、路径文字编辑、演示播放、云同步、Plus 能力都后移。

## 2. 核心愿景
*   **极致视觉**: 复刻手绘感的锥形曲线分支和沿路径排布的文字。
*   **博赞纯正性**: 中心图像、多色、一个概念单元一分支、曲线渐细、线字等长、留白清晰等规则必须被产品结构性保障。
*   **空间专注**: 引入物理纸张边界（A3/A4），通过物理限制引导清晰、有留白的发散性思考，而非无限拥挤堆叠。
*   **单图成纸**: 一个 `.omm` 对应一张纸；复杂主题通过另开独立 `.omm`、文件夹组织或手动外部链接处理，而不是把多张图塞进同一文件。
*   **AI 消摩擦**: AI 作为代笔人/秘书，优先自动完成关键词建议、中心图生成、导入净化等琐事，但不得静默改变用户语义。
*   **知识库基础设施**: 长期商业闭环从单一编辑器扩展到原子化导图文件的检索、发布、同步、版本与团队管理。
*   **生态开放**: 支持 Mermaid、Excalidraw、Figma 等主流工具的导入导出，并对 AI 友好。

## 3. 功能需求 (Functional Requirements)

### 3.0 MVP 范围

MVP 必须包含：

*   **Agent CLI 编排**: 支持 Gemini CLI / Codex CLI / Claude Code 等 Agent CLI 调用本项目 agent skill，形成“长文本 -> skill -> CLI -> Web 预览”的自动化链路。
*   **agent skill 输入约定**: 支持从长文本生成符合“一个概念单元一分支”的简短层级列表。
*   **CLI 校验与启动**: CLI 接收列表，执行契约校验和容量阈值检查；超限时返回可供 Agent 重试的错误，合格时启动本地 Web 预览。
*   **本地 Web 服务**: CLI 可启动本地预览服务，浏览器端负责真实文本测量、布局求解和只读渲染。
*   **只读渲染预览**: Web 页面展示 A3/A4 横向纸张、有机分支、中心图像、多色编码、路径文字和基础素材。
*   **本地导出**: 支持从本地预览导出 `.omm` 和 PNG 图片。

MVP 暂不包含：

*   `.omm` 可视化编辑
*   拖拽、重组、撤销重做
*   路径文字编辑浮层
*   演示播放
*   云同步、账户、Plus 服务
*   移动端 App
*   Obsidian / Logseq / Notion 双向同步

### 3.1 核心编辑器 (Editor Core)
以下是完整产品方向，不属于 MVP 的能力需要按路线图逐步实现。

*   **物理画布约束**: 
    *   支持标准的 **A3 (420x297mm)** 和 **A4** 横向纸张模式。
    *   不支持无限画布生长；标准有机导图始终在固定纸张规格内完成。
    *   纸张空间是硬约束，系统不得通过越界、重叠、缩小字体或拥挤堆叠来继续添加可见分支。
    *   空间不足时，用户需要精炼概念、删除内容、调整结构、另开独立 `.omm` 或手动添加外部链接。
    *   默认新建画布必须为横向纸张，不提供纵向作为标准有机导图模式。
*   **中心图像优先 (Central Image First)**:
    *   每张思维导图必须从中心图像开始，中心节点默认不是纯文本框，而是承载主题图像与标题的中心构图。
    *   长期严格模式下，中心图像至少包含 3 种颜色。Phase 1 为兼容网络矢量图素材现实，允许受控矢量库或 AI 选择的中心 SVG 为单色；但中心仍必须是图像或明确视觉符号，不能退化为普通纯文字中心。
    *   Phase 1 的 AI SVG 中心图只允许来自产品内硬编码白名单的受控矢量源。CLI 仅传递合格 URL，不下载、不净化、不内联 SVG；浏览器负责异步加载、轻量安全检测和失败后回退到确定性的内置中心图模板。
    *   纯普通文本中心只能作为草稿态或待补图状态，不能作为严格模式合格导图。
*   **智能空间填充布局 (Space-Filling)**:
    *   算法自动感知纸张边界，将主干分支优先引向纸张的四个角落。
    *   **分支长度自适应**: 概念单元决定理想分支长度，纸张空间决定最大可用分支长度，系统动态调整分支以实现“线字等长”的视觉目标。
    *   **可见裁切**: 当概念单元超过可用空间时，文本可被裁切；系统不主动警告或阻断，用户通过视觉结果自行调整字体、概念或结构。
    *   **留白保护**: 布局算法必须把分支间距、文字碰撞率、局部密度作为硬约束，避免画面拥挤。
    *   **非填满优先**: 不以“铺满画布”为目标；当内容过密时，用户应精炼、调整结构或另开新图。
*   **多图与外部链接**:
    *   当前不提供产品内子图下钻、面包屑回溯、颜色继承或引用保护。
    *   Web 端可通过“册子”容器组织多张独立 `.omm`，本地与插件场景通过文件夹组织多张独立 `.omm`。
    *   支持简单外部链接或路径引用；文件移动或重命名导致断链时，由用户自行处理。
*   **跨分支关联工具 (Relationship Links)**:
    *   支持在非父子关系的任意两个节点间建立“关联连线”（箭头曲线）。
*   **数字手绘助手 (Organic Sketch Assistant)**:
    *   **AI 涂鸦生成**: 将关键词转换为透明背景的手绘风格图片。
    *   **手绘感素材库**: 提供预设的图标、符号与箭头素材。
    *   **基础视觉素材**: 免费版提供基础中心图模板、常用符号/图标、箭头、主分支视觉标记、基础云朵/强调符号，并支持本地上传图片。
    *   **MVP AI SVG 配图**: Agent skill 可为中心主题选择受控矢量图库 URL；浏览器加载失败或安全检测失败时，自动使用内容哈希选择的内置模板，不阻塞 CLI 预览启动。
    *   **Plus 视觉增强**: Plus 提供 AI 手绘图、高级中心模板、高级素材库、风格化和批量视觉补全。
*   **关键词优化辅助**:
    *   **智能大写**: 英文关键词默认自动转换为全大写，除非用户显式关闭“严格博赞模式”。
    *   **一概念一分支约束**: 每条分支承载一个认知上的概念单元，而不是机械限制为一个语言学词元。中文复合词、英文概念短语只要表达单一概念即可保留。
    *   **异步关键词建议**: 中文 5 字、英文 3 词仅作为本地启发式建议，不作为硬限制。
    *   **免费/Plus 分层**: 免费版使用本地启发式建议；Plus 版使用 LLM 做语义压缩，减少用户手动精炼成本。
    *   **原文主权**: 系统必须保留 `rawInput`，并提供撤销收敛或查看原始输入的交互。
    *   **语义保护**: AI 收敛不得静默改变用户语义。包含否定、数字、人名、日期、专有名词、因果/条件限定等高风险信息时，必须保护原文或让用户确认后再替换。
    *   **必要状态反馈**: 仅保留必要交互状态，不做评分式、审查式、警告式提示。
    *   **引导词辅助**: 提供“为什么、是什么、哪里、谁、如何”等发散提示词，帮助继续联想。
*   **树状结构管理**: 
    *   支持无限层级的父子节点关系。
    *   自动维护树的完整性。
    *   子节点集合必须保留稳定顺序，不能视为无序集合。
*   **意图式编辑交互 (Primary)**:
    *   **意图式拖拽**: 用户拖拽节点时，系统记录目标侧向、顺序或角度偏好，并由布局求解器重新生成合法几何结果。
    *   **节点操作**: `Tab` (子节点), `Enter` (兄弟节点), `Delete` (删除), `F2` (编辑)。
    *   **拖拽重组 (Re-parenting)**: 拖拽节点重组树结构，线条平滑过渡。
    *   **顺序调整**: 支持调整兄弟节点顺序，以服务布局优先级、表达顺序和后续动画编排。
    *   **不允许盒状节点**: 标准有机导图模式下，节点文字必须骑在线条上，不使用矩形、圆角胶囊或圆圈包裹关键词。
    *   **自由导出出口**: 产品内不做自由模式，`.omm` 不保存自由布局或非博赞编辑状态。自由编辑通过导出外部工具可识别的矢量格式完成。
*   **智能排版辅助 (Secondary)**:
    *   **一键局部对齐**: 针对选中分支，自动优化子节点的角度分布。
    *   **全局空间优化布局**: 按照有机布局算法，结合纸张边界自动分配分支位置。
    *   **自动纠偏**: 当手动拖拽破坏博赞结构时，系统应自动回正，而不是长期保留非博赞几何结果。
    *   **空白线条**: 支持插入未命名的空白分支，作为联想占位符。
    *   **云朵边界 (Clouding)**: 支持为完整分支添加云状包围边界，用于组块化和整体记忆。
    *   **BOIs**: 支持对主分支或关键分支添加基本排序理念标签；BOI 是内容组织概念，不等同于节点 ID 或演示顺序。

### 3.2 有机渲染引擎 (Organic Rendering)
*   **极致矢量输出**: 基于 SVG 架构，确保 A3 打印绝对锐利。
*   **锥形分支 (Tapered Branches)**: 从根部到末梢由粗变细。
*   **动态贝塞尔曲线**: 自动计算控制点，确保形态优美。
*   **文字沿路径 (Text-on-Path)**: 文字顺着分支曲线切线方向排布。
*   **分离式文字编辑层**: 呈现时使用路径文字，编辑时使用临时水平输入浮层；输入内容实时同步到路径预览。
*   **色彩继承**: 子节点默认继承父节点的分支颜色。
*   **多色编码**: 颜色系统必须支持按主分支进行显著区分，形成稳定视觉记忆锚点。
*   **图文混排**: 渲染层必须原生支持词语、符号、图像在分支上的混合排布。
*   **受控随机性**: 每张图生成稳定的有机种子，用于驱动分支曲率、锥形比例、轻微抖动和手绘感差异。
*   **禁止机械直线**: 标准有机导图模式下不提供等宽直线分支作为默认渲染。

### 3.3 数据与兼容性 (Data & Interoperability)
*   **单文件单图**: 一个 `.omm` 文件对应一张思维导图。
*   **原子文件优先**: `.omm` 是独立可读写的原子文件，不依赖专有数据库或黑盒包格式承载核心内容。
*   **多图组织**: Web 端以“册子”管理多张图，本地和插件环境以文件夹管理多张图。
*   **资源自包含**: `.omm` 必须保证独立打开后图片完整；必要图片资源随 `.omm` 保存，外部路径只能作为来源记录或可选引用。
*   **简单外部链接**: 外部链接使用路径引用即可，不提供引用保护、断链检测、自动修复或重命名重构作为 MVP 能力。
*   **多格式导入/导出**: 支持 Mermaid, Excalidraw, Figma, PNG, SVG, JSON, PDF (A3/A4 规格)，并支持从导图语义派生 Mermaid 二级/多级列表作为辅助导出。
*   **导入净化管线**: 导入的非博赞结构应被重塑为一张新的博赞导图，包括中心图像工作流、概念单元、颜色、分支形态、线字等长和留白；不保留源结构、源对象映射或源文件快照。
*   **冲突解决**: 对层级过深、空间不足等无法自动净化的内容，提供 AI 收敛、手动精炼、调整结构、另开新图或外部链接的处理入口。
*   **自由再编辑出口**: 由于产品内部采用严格博赞模式和自动纠偏，用户应可将结果导出为带语义化图层和内嵌素材的 SVG，供 Figma、Photoshop 等外部工具继续自由排版、标注和二次创作。

## 4. 用户体验 (User Experience)
*   **渲染性能**: 针对 SVG 节点优化。
*   **无缝编辑**: 双击路径文字后弹出水平编辑浮层，按 Enter 或失焦后平滑回贴到曲线。
*   **演示播放**: 支持单一用户自定义讲解路径。每按一次“下一页”，展开一个节点或该节点的子节点；演示顺序由 `NodeId[]` 或 step 列表驱动，不等于 BOI，也不等于树结构顺序。
*   **复习机制**: 复习与快速重绘检查不进入当前阶段，延后到移动端 App 开发之后再评估。
*   **创意保护**: 对非常规、跳跃式、看似荒诞的关键词不做语义纠正压制，避免过度规范化扼杀联想。
*   **个人风格**: 允许用户沉淀个性化配色、图像风格和符号偏好，但不能破坏博赞核心法则。

## 5. 商业与付费边界 (Business Model)
*   **免费核心**: 严格博赞核心编辑、固定纸张导出、基础素材、基础中心图模板和本地启发式关键词建议必须可用（本地图片上传将作为 Phase 2 的免费能力提供）。
*   **Plus AI 效率**: Plus 提供 LLM 关键词收敛、AI 手绘图生成、高级中心图模板、高级素材库、风格化、去背、高清处理和云端资产管理。
*   **Plus 数据服务**: 支持跨 `.omm` 文件的语义检索、RAG 问答、自动版本历史、冲突检测和团队共享库；这些服务元数据存放在云端，不改变 `.omm` 开放格式。
*   **Plus 发布能力**: 支持一键生成可缩放、可密码保护的 Web 交互分享页；Web 分享页保持 A3/A4 纸张比例。
*   **长期集成方向**: Obsidian、Logseq、Notion 等双向同步插件属于后期路线，当前只保留导入/导出或文件级兼容表述。

## 6. 技术约束 (Technical Constraints)
*   **前端框架**: Vue 3 (Composition API) + TypeScript。
*   **状态管理**: Pinia + 响应式树模型。
*   **渲染技术**: 纯 SVG。
*   **构建工具**: Vite。
*   **移动预留**: 架构层预留压感笔轨迹采样、平滑化和手绘输入接口，但移动/平板作为 Phase 5+ 长期路线图。
