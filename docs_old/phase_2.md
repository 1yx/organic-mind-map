# Phase 2: 生成式整图合成与可编辑画布

## 为什么 Phase 2 不能只继续优化分支算法

GPT Image 2 生成的手绘思维导图在最终观感上明显强于当前项目，而且强的不是一点点。这个差距并不是单一的“分支算法更好”造成的，而是因为它做的是**整张图的生成式视觉合成**，而当前项目做的是**结构化数据到确定性 SVG renderer**。这两件事的能力边界不同。

当前 renderer 的路径更接近：

```text
branch geometry -> text placement -> collision check
```

生成式整图合成的路径更接近：

```text
idea -> composition -> branch/text/image simultaneously
```

Phase 2 需要承认这个差异：如果目标是商业上可感知的高质量手绘思维导图，仅仅继续打磨确定性分支几何是不够的。

## GPT Image 2 效果更好的原因

### 1. 整体构图是全局优化的

中心、主分支、子分支、插图、文字位置是一起生成的，不是先排树再补图标。它像人在一张纸上“画完整画面”。

这意味着它天然会同时考虑：

- 中心图的位置和视觉重量
- 主分支的粗细、方向和留白
- 子分支从主干生长的位置
- 文字和插图之间的相互避让
- 画面整体的视觉节奏

### 2. 文字和分支天然贴合

生成图里的文字不是独立 label 放上去，而是跟着枝条走势、视觉节奏一起安排。文字、分支和插图在视觉上是一个整体。

当前确定性 renderer 更容易出现的问题是：先生成 branch，再把 text 当作附着物摆放，最后通过碰撞检测修补。这种流程可以验证、可复现，但容易显得机械。

### 3. 图像语义很丰富

生成图可以为每个叶子概念生成图像隐喻，例如火箭、秒表、团队、盾牌、机器人、图表等。当前 renderer 最多支持 branch marker 或 center visual，还没有“每个概念生成视觉符号”的能力。

这会直接影响商业感知：用户第一眼看到的是一张有插图、有记忆锚点、有手绘表达的图，而不是一棵被渲染出来的结构树。

### 4. 层级表达更自然

生成图里的主分支粗、子分支细，子分支从主干分段长出，文字沿外侧排布。这比当前 demo 中“几何规则分叉”的效果更接近真实手绘思维导图。

确定性 renderer 可以逐步追这个效果，但如果只从几何规则出发，会很容易陷入局部优化：分支角度更自然了，但整张图仍然不像人画的。

### 5. 它接受不精确

生成图里的文字可能有瑕疵，结构不完全可编辑，对象也不可复用，但视觉上非常像人画的。当前项目追求可验证、可导出、可重开、可结构化，所以会更僵硬。

这不是谁对谁错，而是产品目标不同：

- 生成式图片优先追求最终观感和商业吸引力。
- 结构化 renderer 优先追求可编辑、可验证、可复现和长期资产化。

## Phase 2 的产品判断

Phase 2 不应把目标定义为“让确定性 SVG renderer 立刻达到 GPT Image 2 的观感”。更合理的目标是建立混合路线：

```text
YAML / OrganicTree
  -> GPT Image 2 生成高质量手绘整图
  -> 参考图 + YAML 重建可编辑 canvas
  -> 用户可修正文字、移动对象、替换素材、导出结果
```

也就是说：

- YAML / OrganicTree 继续作为结构真值。
- GPT Image 2 用于生成商业上足够吸引人的整图。
- 可编辑 canvas 用于修正、复用和资产化。
- 确定性 renderer 继续作为长期结构化能力，而不是阻塞商业 MVP 的唯一出口。

## Phase 2 的优先级

Phase 2 应优先验证以下方向：

1. **生成式整图质量**：用 YAML 约束 GPT Image 2，稳定生成符合有机思维导图规则的手绘图。
2. **可编辑重建**：用 YAML 保证结构正确，用生成图作为视觉参考，重建可编辑 canvas。
3. **素材提取**：从生成图中裁切插图素材，去背景后贴回可编辑画布。
4. **局部修正体验**：用户可以改错字、移动标签、替换插图、调整分支。
5. **结构化保存**：保存 YAML / OrganicTree、canvas 对象状态、生成图引用和提取素材。

这个阶段的核心不是追求像素级 1:1 复刻，而是验证一个商业上成立的体验：

```text
生成图足够好看，结构足够可控，用户能改关键错误。
```

## 与 Phase 1 的关系

Phase 1 的确定性 SVG renderer 仍然有价值，它提供：

- 严格 OrganicTree contract
- CLI validation
- 可复现 preview
- `.omm` 结构化导出
- 长期可编辑基础

但 Phase 2 需要在此基础上增加生成式视觉能力。确定性 renderer 不应该成为高质量手绘输出的唯一实现路径。
