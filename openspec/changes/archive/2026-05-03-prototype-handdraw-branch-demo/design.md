## Context

The current renderer already supports curved paths, tapered filled branches, and browser-side SVG preview. The visual result is still too mechanical because branch bodies are generated as straight-line tapered outlines and child branches start from rigid endpoints. The reference images in `fixtures/handdraw/` show a different target: organic branches that curve outward while preserving mostly horizontal reading segments.

This change creates a demo surface before changing production renderer behavior. The demo is a visual spike used to compare branch-generation approaches and decide whether the algorithm is suitable for a later renderer rewrite.

## Goals / Non-Goals

**Goals:**

- Render a standalone branch-only SVG demo under `handdraw-branch-demo/`.
- Show readability-biased branches: outward organic growth with near-horizontal terminal segments.
- Generate branch bodies from sampled variable-width ribbons rather than straight-line quadrilaterals.
- Add deterministic edge jitter to approximate hand-drawn branch contours.
- Place child branch anchors along parent branches instead of forcing all children to share one endpoint.
- Capture Playwright screenshots for visual review.

**Non-Goals:**

- Do not modify `@omm/renderer` production output.
- Do not render branch text in the demo.
- Do not change `OrganicTree`, `.omm`, CLI, export, or Web preview behavior.
- Do not introduce a full layout solver or force-directed graph layout.
- Do not use raster/canvas-only output for the branch shape.

## Decisions

### Use a Static Demo Under `handdraw-branch-demo/`

The prototype SHALL live under `handdraw-branch-demo/` so it is explicitly separated from production code and generated artifacts. This keeps the experiment easy to open in a browser while avoiding accidental coupling to the renderer package.

Alternative considered: add a Vite route to `@omm/web`. Rejected because this would mix exploratory UI with the MVP preview app.

### Use Readability-Biased Cubic Splines

Each branch centerline will be a cubic Bezier curve with separate growth and reading directions. The curve may leave the center at an organic angle, but its terminal tangent should be near-horizontal on the left or right side.

Alternative considered: fully radial curves. Rejected because the reference images prioritize readable text placement over pure radial geometry.

### Use Sampled Variable-Width Ribbons

Branch bodies will be generated by sampling the centerline, computing normals, and building a closed outline from left/right edge points. Width decays from root to tip with an easing function.

Alternative considered: single SVG stroke with variable `stroke-width`. Rejected because SVG strokes cannot taper continuously along one path without masks or custom outlines.

### Use Deterministic Edge Jitter

The demo will apply seeded low-amplitude normal-direction jitter to branch edges. The centerline remains smooth so future text can follow stable paths.

Alternative considered: jitter the centerline heavily. Rejected because it would make future text placement hard to read.

### Use Parent-Curve Anchors for Children

Child branches will anchor at configurable `t` values along the parent curve, usually in the parent branch's latter half. This better matches hand-drawn examples where child branches grow out of the parent limb rather than from a single endpoint.

Alternative considered: all children start at the parent endpoint. Rejected because it creates starburst clusters and cramped labels.

## Risks / Trade-offs

- Hand-drawn jitter can create self-intersecting outlines -> keep jitter amplitude small and depth-scaled.
- Near-horizontal terminal segments can make branch spacing harder -> demo should show several branch counts and expose parameters for later tuning.
- A demo under `handdraw-branch-demo/` is not production-tested -> keep implementation small and Playwright-verified, then use a later change for renderer integration.
- Branch-only validation cannot prove text readability -> this change intentionally defers text until the branch shape is accepted.
