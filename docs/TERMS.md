# Organic Mind Map Terms

## Canonical Branch Data

`canonical branch data` is the product's standard source-of-truth representation for editable branch geometry.

It is not a debug artifact, not a mask, not a temporary SVG export, and not a Paper.js internal object. It is the structured branch data that the editor saves, loads, renders, edits, exports, and later uses as Phase 3 training/correction data.

## Content Outline

`content outline` is the semantic structure provided by the user (via simple YAML) or generated by an AI agent (via LLM) before visual reconstruction.

It describes what the mind map should say, not how the final map should look.

In Phase 2, the preferred format for manual input is a **Simple YAML Outline**: an indentation-based hierarchy without explicit field names.

Example:

```yaml
Anthropic 产品之道
  极速交付
    研究预览
    跨职能
```

Interpretation:
- Line 1 (indent 0): Center Visual/Text
- Line 2+ (indent 1+): Main Branches and their recursive subbranches

The content outline is used to:
- guide GPT-Image-2 generation
- classify OCR text
- align extracted text with known concepts
- build canonical `branch` and `subbranch` objects
- detect doodle-local text and `unassigned_text`

## Branch Outline

`branch outline` is the closed visual boundary of a rendered branch.

It is generated from:

```text
centerline + widthProfile
```

Rule:
```text
canonical branch data stores centerline + widthProfile
renderer generates branch outline
```

## Branch Schema

In Organic Mind Map, a `branch` is the top-level same-color branch system that grows from the center.
A `subbranch` is any descendant branch that grows from a branch or another subbranch.

Top-level branch example:
```json
{
  "id": "branch_001",
  "class": "branch",
  "concept": "极速交付",
  "color": "#fc8b05",
  "thicknessScale": 1.0,
  "mainStroke": {
    "centerline": "M 620 500 C 540 420 460 310 330 250",
    "widthProfile": {
      "type": "root-to-tip-gradient",
      "rootWidth": 38,
      "tipWidth": 12
    }
  },
  "children": ["subbranch_001_001", "subbranch_001_002"]
}
```

## Width Profile

`widthProfile` defines how the visible branch width is derived from the editable centerline.
Phase 2 uses gradient/tapered width:
```json
{
  "type": "root-to-tip-gradient",
  "rootWidth": 38,
  "tipWidth": 12
}
```

## Centerline

`centerline` is the editable path running through the middle of the visible branch.
Centerline direction is canonical:
```text
M point = root side (near center/parent)
path end point = tip side (outer edge)
```
Current preferred representation is an SVG path string.
