# Business Plan: Organic Mind Map

## 1. 产品定位与核心价值
**一句话定义**: 开源的博赞有机思维导图工具，免费提供纯正的有机导图创作能力，围绕 AI、云服务和高级创作增强收费。

### 核心差异化
*   **品类聚焦**: 不做泛白板，不做大而全，专注“博赞式有机思维导图”。
*   **开源策略**: 核心编辑器开源，用产品纯度和社区传播获客。
*   **分层收费**: 博赞法则相关的基础能力必须免费，付费只针对效率增强、内容增强和组织能力。
*   **MVP 先验证 Agent 编排链路**: 首版不做可视化编辑器，先通过 Gemini CLI / Codex CLI / Claude Code 等 Agent CLI 调用 skill，再由本项目 CLI 校验并启动本地 Web 预览，最终在浏览器端导出 `.omm` 与 PNG，验证从长文本到有机导图成品的最短路径。
*   **生态融合**: 兼容 Mermaid、Figma、Photoshop 等生态，扩大导入导出入口；Obsidian / Logseq / Notion 深度同步属于后期方向。
*   **自由出口**: 产品内部保持严格博赞模式，但允许用户把结果导出为带语义化图层的 SVG，在 Figma、Photoshop 等工具中进一步自由编辑。
*   **文件模型简洁**: 一个 `.omm` 只对应一张图，多图组织交给 Web 册子或本地文件夹。

## 2. 商业原则

### 2.1 什么不能收费

凡是为了保证“这是一张合格的博赞有机思维导图”所必需的能力，都不能作为付费墙，包括：

*   横向纸张与边界感
*   中心图像工作流
*   曲线渐细分支
*   线字等长
*   一个概念单元一分支约束
*   英文关键词默认大写
*   多色主分支编码
*   关联线
*   云朵边界
*   空白分支
*   基础视觉素材与本地图片上传
*   本地保存与基础导出

### 2.2 什么适合收费

适合收费的不是“正确性”，而是“效率、质量、发布与规模化”：

*   AI 自动生成与理解
*   手绘风格图片与素材增强
*   云同步与跨设备
*   历史版本与恢复
*   演示、汇报、发布能力
*   归档管理与企业安全

我们采用 **"Open Core + SaaS Plus"** 模型：

| 维度 | 开源核心 (Community) | 付费增值 (Plus - 预计 $6-8/月) |
|--|-----------|------|
| **目标用户** | 个人用户、学生、研究者、笔记用户、自托管用户 | 深度创作者、内容生产者、研究型个人用户、小型机构 |
| **功能范围** | Agent CLI 调用 skill 文本切分、CLI 校验与本地 Web 预览、浏览器端 `.omm`/PNG 导出；后续扩展到完整编辑 | 云同步、AI 生成、手绘素材增强、演示、版本历史、发布归档 |

## 4. 开源与付费功能划分

### 4.1 开源免费 (Community / MIT)
*   **博赞核心编辑器**: 中心图像、多色分支、锥形曲线、文字沿路径、线字等长、一个概念单元一分支约束。
*   **基础结构能力**: 关联线、空白分支、云朵边界、BOI 标签。
*   **基础视觉能力**: 基础中心图模板、常用符号/图标、箭头、主分支视觉标记、本地图片上传。
*   **本地优先**: `.omm` 单文件单图保存与加载，必要图片资源随 `.omm` 自包含保存，配合文件夹组织多图。
*   **基础导出**: PNG、带语义化图层的 SVG、JSON、PDF、Mermaid 二级/多级列表。
*   **基础导入**: Mermaid、Excalidraw 导入，并生成新的博赞导图。
*   **自由导出出口**: 支持导出高质量 SVG，作为严格博赞模式之外的后续加工路径。
*   **集成能力**: 提供 `@organic-mindmap/core` npm 包，允许第三方应用嵌入。

### 4.2 Plus 个人版
*   **云同步**: 跨设备访问、自动保存、云备份。
*   **版本历史**: 历史快照、回滚恢复。
*   **AI 结构生成**: 文本、文章、网页、PDF、播客内容自动生成思维导图初稿。
*   **AI 关键词压缩**: 在不改变用户语义的前提下，将句子压缩为更符合博赞原则的概念单元。
*   **手绘风格图片**: 为中心图像和关键节点生成手绘风格图片。
*   **高级素材库**: 高质量图标、符号、箭头、中心图模板、风格化与批量视觉补全。
*   **云端资产管理**: 图片云端管理、高清处理、去背、跨设备复用。
*   **演示模式**: 用户自定义讲解路径，逐节点或逐子树展开。

### 4.3 Enterprise 企业版
*   **SSO / SCIM / 审计日志**
*   **私有部署或专属区域**
*   **高级安全与合规**
*   **企业支持与 SLA**

## 5. 为什么“手绘风格图片”适合付费

这类能力不决定导图是否合格，但显著提升导图的完成度和传播价值，适合做付费增强：

*   它提升的是视觉表现力，不是基础可用性。
*   它依赖 AI 生成、素材库建设或版权采购，边际成本更高。
*   它对内容创作者、课程讲师、咨询顾问、知识博主的价值明显高于普通用户。
*   免费版只需保证“能使用基础视觉资源完成中心图像”，付费版再提供“更自由、更好看的图像能力”。

因此建议：

*   **免费**: 提供基础符号素材、基础中心图模板、主分支视觉标记、本地图片上传，以及部分沉淀下放的预制图像模板。
*   **付费**: AI 手绘图生成、高级素材库、风格化中心图模板、去背、高清处理、云端资产管理和批量视觉补全。

## 7. 发展路线图 (Roadmap)

### Phase 1: MVP & 社区引爆 (1-2个月)
*   发布 agent skill，用于被 Gemini CLI / Codex CLI / Claude Code 等 Agent CLI 调用，将长文本切成符合博赞要求的简短层级列表。
*   发布 CLI，接收 agent skill 输出，执行契约校验和容量阈值检查。
*   内容过多时，CLI 返回可被 Agent 捕获的错误，要求重新生成更精简列表。
*   校验通过后，CLI 启动本地 Web 服务；生产路径仅托管预构建静态资源和只读 `/api/document`，不启动前端 Dev Server，不做文件监听或热重载；浏览器端完成文本测量、布局和只读预览。
*   支持在浏览器端导出 `.omm` 与 PNG 图片。
*   暂不做 `.omm` 可视化编辑器，先验证渲染质量、纸张约束和长文本到导图的工作流。

### Phase 2: 云化与增强 (2-3个月)
*   打磨 SVG / PDF / Mermaid 等基础导入导出。
*   逐步补齐可视化编辑能力。
*   上线 Cloud Sync。
*   发布版本历史。
*   发布 AI 自动生成导图内测。
*   上线手绘风格图片与高级素材包。

### Phase 3: 商业化与生态 (长期)
*   开启 Plus 订阅。
*   完善企业级权限与部署能力。
*   构建第三方主题、素材和插件市场。

## 8. 风险与控制点

*   **免费版过弱**: 如果核心能力被过度阉割，用户不会认可“最纯正的有机导图工具”定位。
*   **免费版过强**: 如果 AI、云同步、高级素材增强和云端资产管理全部免费，商业转化会过弱。
*   **AI 成本过高**: 图像生成和结构生成都有推理成本。应优先按订阅额度、积分包或自带 Key 模式控制。
*   **产品定位跑偏**: 不能为了商业化把产品做成泛白板或普通脑图工具。

建议的控制原则：

*   免费版解决“能不能正确画”。
*   Plus 解决“能不能更快、更好看、更省力”。
*   Enterprise 解决“能不能可部署、可管理、可治理”。

**副标题**: Community 提供完整的有机导图基础能力。Plus 解锁 AI、云同步和高级视觉增强。Enterprise 面向部署、安全和组织管理。

### 9.3 页面上的关键说明

建议在定价页显式写出以下信息，避免用户误解：

*   **博赞核心能力永久免费**：不会把中心图像、曲线分支、一个概念单元一分支等基础能力锁进付费版。
*   **Plus 卖的是增强，不是阉割解锁**：付费购买的是 AI、云服务、素材质量和演示能力。
*   **可先免费使用，再按需升级**：降低首次尝试门槛。
*   **支持年付优惠**：鼓励内容创作者和知识工作者长期订阅。

### 9.4 推荐的定价页 FAQ

**Q: 免费版是不是功能残缺？**

A: 不是。免费版已经包含画出合格博赞有机思维导图所需的全部核心能力。

**Q: 为什么手绘风格图片要收费？**

A: 因为它属于视觉增强能力，依赖 AI 推理和高质量素材体系，不影响基础导图是否成立。

**Q: 为什么没有团队协作版？**

A: 当前产品聚焦严格博赞模式下的单人创作体验，不把实时协作作为核心方向。商业化主要围绕 AI、云同步、素材增强、部署与安全展开。

**Q: 可以自托管吗？**

A: 可以。开源核心支持自托管和二次集成；企业级部署能力可单独报价。

### 9.5 定价页转化策略

*   默认突出 `Plus`，因为它最适合作为个人用户的自然升级点。
*   `Community` 必须写清“永久免费核心能力”，这是信任入口。
*   `Enterprise` 只保留关键信息，把复杂条款留给销售沟通。
